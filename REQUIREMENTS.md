# 実況ブラウザ LiveFakeTauri - 要件定義書

## 概要

Tauri（Rust バックエンド + JavaScript フロントエンド）ベースの日本語掲示板専用実況ブラウザ。  
**対応OS: Windows 10 / Windows 11（64ビット）のみ。Linux・macOS は対象外。**  
対応サイト: 5ch.io / したらば / JPNKN  
Portable 形式（EXE 同梱フォルダで自己完結）で配布する。

---

## 1. UI・レイアウト

### 1.1 メイン画面構成

- **ツールバー**: URL入力欄、「開く」ボタン、「再読込」ボタン、検索欄、設定ボタン、字幕トグルボタン
- **左パネル（ツリー）**: ボード一覧 / スレッド一覧 / ボードカタログ（ブックマーク）の切り替え表示
- **タブバー**: 複数スレッド・ボードを同時表示（デフォルト最大20タブ）
- **メインコンテンツエリア**: レス（投稿）の表示
- **新着パネル**: 最新レスをリアルタイムで表示するサブペイン
- **ステータスバー**: 最終更新時刻とステータスメッセージ

### 1.2 リサイザー

- 左パネル：左右ドラッグでカラム幅を変更（160px ～ 700px）
- 新着パネル：上下ドラッグで高さを変更（80px ～ 420px）
- リサイズ後のサイズは設定に自動保存し、次回起動時に復元する

### 1.3 タブ操作

- クリックでタブ切り替え
- ドラッグ＆ドロップでタブ順序変更
- ×ボタンでタブクローズ
- `Ctrl+Tab` / `Ctrl+Shift+Tab` で隣接タブへ移動

### 1.4 テキスト表示設定

- デフォルトフォント: MS UI Gothic, Meiryo, Segoe UI Emoji, Noto Color Emoji
- フォントサイズ: 10 ～ 28px（デフォルト 14px）
- レス間隔: 0 ～ 40px（デフォルト 10px）
- スムーススクロール: 設定で有効/無効の切り替え

### 1.5 キーボードショートカット

| キー | 動作 |
|------|------|
| `Enter` / 「開く」ボタン | URL 読み込み |
| `Ctrl+F` | 検索欄フォーカス |
| `Ctrl+Tab` | 次のタブへ |
| `Ctrl+Shift+Tab` | 前のタブへ |
| `Escape` | モーダル/コンテキストメニューを閉じる |

## 2. BBS 読み込み・表示機能

### 2.1 対応サイト

#### 2.1.1 5ch.io 系

- 対応ドメイン: `*.5ch.io`, `*.5ch.net`, `*.2ch.net`
- ホスト正規化: `mao.5ch.io`, `eagle.5ch.io` 等 → `{server}.5ch.io` に統一
- スレッド一覧取得: `subject.txt`
- レス取得: `*.dat` 形式
- HTTP → HTTPS 自動アップグレード

#### 2.1.2 したらば（jbbs.shitaraba.net）

- エンコーディング: EUC-JP
- スレッド一覧取得: `subject.txt`
- レス解析: HTML 形式（DT/DD タグペア）
- メール欄の HTML メタデータ抽出対応

#### 2.1.3 JPNKN（bbs.jpnkn.com）

- エンコーディング: Shift_JIS
- データ取得: JSON API（jpnkn.com API）
- レス解析: dat 形式

### 2.2 マルチエンコーディング自動検出

- 対応: UTF-8, Shift_JIS, EUC-JP
- 自動スコアリングで最適なエンコーディングを選択
  - 置換文字（U+FFFD）数 × -10
  - 日本語（ひらがな・カタカナ・漢字）含有で +20
  - `<>` 含有で +5

### 2.3 HTML 解析

- HTMLエンティティ変換: `&nbsp;`, `&amp;`, `&lt;`, `&gt;`, `&quot;`, `&#39;`, `&#xHEX;`, `&#DEC;`
- タグ変換: `<br>` → 改行 / `</p>`, `</div>`, `</dd>`, `</dt>` → 改行 / その他タグ除去

### 2.4 レスのメタデータ抽出

- 投稿者情報: 名前 / メール / 日付 / ID
- ID 抽出: 正規表現 `ID:[A-Za-z0-9+/._-]+`
- スレッドタイトル: ページタイトルから抽出

### 2.5 5ch BBS Menu

- URL: `https://menu.5ch.io/bbsmenu.json`
- 全掲示板リストをカテゴリー構造で表示
- カテゴリーの展開/折畳み対応
- ダブルクリックで該当ボード読み込み
- キャッシュ: `bbs-menu.json`

### 2.6 外部板登録

- 任意のボード URL と名前を登録（`+ 外部板` ボタン）
- 登録ボードは `external-boards.json` に永続保存
- 登録後 `__external__` カテゴリを自動展開して表示
- BBS Menu 未読込状態でも外部板は表示可能
- ボードの削除はコンテキストメニューから実行

### 2.7 自動再読込

- インターバル: 15 ～ 300秒（デフォルト 15秒）
- ON/OFF トグル、手動再読込ボタン
- 前回の最終レス番号を記録し差分のみ取得

### 2.8 自動スクロール

- 新着レス受信時に自動でスクロール
- 設定で有効/無効の切り替え

## 3. 書き込み（投稿）機能

### 3.1 フォーム入力項目

- 名前（FROM）
- メール（mail）: 空欄時は `sage` 等を指定可能
- 本文（MESSAGE）
- スレッド立て時のみ: タイトル（subject）

### 3.2 事前バリデーション

- 本文空欄チェック
- タイトル空欄チェック（新スレ立て時のみ）

### 3.3 サイト別投稿実装

#### 3.3.1 5ch 系

- 送信先: `https://{server}.5ch.io/test/bbs.cgi`
- Referer: `https://{server}.5ch.io/test/read.cgi/{board}/{thread_key}/`
- エンコーディング: Shift_JIS URL エンコード
- 投稿フロー: POST → 確認画面 → 再 POST → 成功
- MonaTicket 自動管理（取得・再取得）

#### 3.3.2 したらば

- 送信先: `https://jbbs.shitaraba.net/bbs/write.cgi`
- エンコーディング: EUC-JP URL エンコード
- 成功判定: `write_done.cgi` へのリダイレクト検出 または 「書き込みが完了しました」テキスト検出

#### 3.3.3 JPNKN

- 送信先: `https://bbs.jpnkn.com/test/bbs.cgi`
- エンコーディング: Shift_JIS URL エンコード

### 3.4 HTML レスポンス解析

解析優先順: `<!-- 2ch_X:xxx -->` コメント → `<title>` タグ → `<b>` タグ内テキスト

| ステータス | 判定条件 |
|-----------|---------|
| Success | `<!-- 2ch_X:true -->` |
| Confirmation | `<!-- 2ch_X:cookie -->` または `<!-- 2ch_X:check -->` |
| Regulation | 「規制」「連投」「sec たたないと」「プロバイダ」等キーワード |
| ServerBusy | 「お茶でも飲みましょう」 |
| ThreadClosed | 「スレッドストップ」「もう書けません」「1001」 |

### 3.5 確認画面の Hidden Fields 自動処理

- 確認画面の `<input type="hidden">` を全て自動抽出
- 次の POST 時に自動付与

### 3.6 スレッド立て機能

- 新スレ作成（タイトル + 本文必須）
- 成功時に新スレッドキーを返却

## 4. Cookie 永続管理

- 保存ファイル: `cookies.json`（JSON 形式）
- 保存情報: Cookie 名/値、ドメイン、有効期限（UNIX タイムスタンプ）、発行元 User-Agent
- Set-Cookie ヘッダ自動解析: `expires` 形式および `max-age` 形式対応
- 有効期限切れ Cookie の自動フィルタリング
- 5ch 系: 発行元 UA 不一致の Cookie をフィルタリング
- MonaTicket 失敗時: 自動削除して 1 回のみ再取得試行

## 5. 画像表示機能

### 5.1 画像取得とプレビュー

- 画像を Base64 エンコードして data URL 化（`fetch_image`）
- 新しいウィンドウでポップアップ表示（`open_image_popup`）
  - ウィンドウサイズ自動調整（最小 200×150、最大スクリーンサイズ-マージン）
  - 背景色: デフォルト `#1a1a1a`

### 5.2 画像 URL 置換ルール（ImageViewURLReplace.txt）

- フォーマット: TSV（タブ区切り）
  ```
  検索正規表現<TAB>置換文字列<TAB>リファラ（省略可）
  ```
- デフォルトルール:
  - Twitter/X の画像 URL 正規化（pbs.twimg.com）
  - ニコニコ動画サムネイル変換（nicovideo.jp → CDN 直接取得）
- カスタムルールをユーザーが追記可能
- リファラ指定時は画像取得時に Referer ヘッダを付与

### 5.3 画像保存

- 保存先フォルダをファイルダイアログで選択（選択結果をキャッシュ）
- ファイル名: URL から自動抽出、提示なければ `image_YYYYMMDD_HHMMSS.{ext}` 生成
- MIME タイプ自動判定: jpg / png / gif / webp / bmp

### 5.4 Image Popup ウィンドウの実装方式

- AppHandle の State に data URL を事前保存（XSS 防止のため eval() 不使用）
- ウィンドウ生成後に IPC で `get_image_popup_data` を呼び出し
- データはワンタイム利用（メモリから即時削除）

### 5.5 画像アップロード（tadaup.jp）

- アップロード先: `https://tadaup.jp/wp-json/custom/v1/upload`
- ファイル選択はブラウザの `<input type="file">` から実行
- 複数ファイル同時アップロード対応
- アップロード結果にサムネイル URL・ページ URL を表示
- 履歴: 最新 100 件を `upload_history.json` に永続保存
- 履歴パネルから過去のアップロード一覧を参照可能

## 6. 字幕機能

### 6.1 字幕ウィンドウ

- 外観: 背景色 `rgba(26, 26, 46, 1.0)` デフォルト濃紺、テキスト色 `#ffffff`
- フォント: MS UI Gothic, Meiryo, 絵文字対応
- ドラッグ可能領域あり（`data-tauri-drag-region`）

### 6.2 表示要素

- `#subtitleMeta`: スレッドタイトル / 投稿者名 / ID / 日付（小フォント）
- `#subtitleBody`: 本文（大フォント、複数行折り返し）

### 6.3 フォント設定

| 項目 | 範囲 | デフォルト |
|------|------|-----------|
| 本文フォントサイズ | 10 ～ 96px | 28px |
| メタ情報フォントサイズ | 8 ～ 48px | 12px |
| 行間 | - | 1.4 |

### 6.4 テキストハイライト

- 指定テキストを背景色で強調表示
- `<span class="hl-text" style="background:#{color}">` 形式で実装
- UI から色を変更可能

### 6.5 ID 色分け

- 投稿者 ID を指定色で表示
- `<span class="hl-id" style="color:#{color}">` 形式
- 日別キャッシュ（前日以前のデータは自動削除）

### 6.6 ウィンドウ管理

- **背景透明度**: 0.1 ～ 1.0（リアルタイム変更）
- **Always on Top**: トグル可能
- 初回表示時に自動生成、非表示/表示は Show/Hide 切り替え

## 10. 音声読み上げ機能

### 10.1 SAPI（Windows 標準）

- インストール済みボイス一覧取得
- ボイスインデックス指定
- 速度: -10 ～ +10
- 音量: 0 ～ 100
- 実装: Windows COM インターフェース（STA スレッドで実行）

### 10.2 棒読みちゃん（RemoteTalk API）

- RemoteTalk.exe パス指定（ユーザー選択）
- 速度・トーン・音量・ボイス ID は棒読みちゃん本体側で設定（UI には表示しない）

### 10.3 VOICEVOX

- デフォルトエンドポイント: `http://127.0.0.1:50021`
- スピーカー ID（キャッシュ）
- 速度スケール: 0.5 ～ 2.0
- ピッチスケール: -0.15 ～ +0.15
- イントネーションスケール: 0.0 ～ 2.0
- 音量スケール: 0.0 ～ 2.0
- Rust 側で WAV を生成し base64 エンコードして返却、フロントエンドで Blob URL を生成して `<audio>` 再生

### 10.4 読み上げ省略（文字数制限）

- 指定文字数を超えるレスは読み上げを省略する
- 設定範囲: 10〜300文字（10文字刻み推奨）
- 「無制限」オプションあり（全文読み上げ）
- デフォルト: 無制限
- 設定キー: `Speech.maxReadLength`（0 = 無制限）
- 省略時の動作: 指定文字数まで読み上げた後、末尾に「長文のため以下省略」を付与して読み上げ
- res 1001・1002 は TTS 読み上げをスキップする（板のシステムメッセージのため）

### 10.5 レス番号右クリックから読み上げ開始

- レス番号を右クリックすると「このレスから読み上げ」メニューを表示
- 選択すると、現在読み上げ中の音声を即座に中止（SAPI: Stop / VOICEVOX: 再生停止）し、指定レスから順番に読み上げを開始
- 読み上げ対象: 指定レス以降の全レス（NG 非表示レスはスキップ）
- 読み上げ中に新着レスを取得した場合、キューに追加して続行

## 12. HTTP クライアント

| クライアント | 用途 | タイムアウト |
|-------------|------|------------|

- 保有期間: 1 ～ 365日（デフォルト 7日）
- 自動削除: 設定日数より古いファイルを起動時に削除

## 16. レス表示領域コンテキストメニュー

レス表示領域で右クリックした際にカスタムコンテキストメニューを表示する。  
ブラウザデフォルトメニューは `preventDefault` で抑制する。

### 16.1 メニュー構成

```
┌──────────────────────────────┐
│ 「選択テキスト」をハイライト ▶ │→ 15色サブメニュー（テキスト選択時のみ）
│ ID:xxxxx をハイライト        ▶ │→ 15色サブメニュー（ID検出時のみ）
│ 名前「xxx」をハイライト      ▶ │→ 15色サブメニュー（名前要素上のみ）
│ ハイライト解除                 │  （対象がハイライト済みの場合のみ）
│ ───────────────────────────── │
│ このレスから読み上げ           │  （レス番号上で右クリック時のみ表示）
│ ───────────────────────────── │
│ スレ全体をコピー               │  全レスをテキスト形式でクリップボードへ
│ ───────────────────────────── │
│ ✓ オートリロード               │  ON/OFF トグル
│   オートスクロール             │  ON/OFF トグル
│ ✓ 読み上げ                     │  ON/OFF トグル
└──────────────────────────────┘
```

### 16.2 表示条件分岐

| メニュー項目 | 表示条件 |
|-------------|---------|
| 選択テキストをハイライト | `window.getSelection()` が空でない |
| ID をハイライト | クリック位置のレス要素から ID を抽出できる |
| 名前をハイライト | クリック位置が名前要素上、またはレス要素から名前を抽出できる |
| ハイライト解除 | 選択テキスト・ID・名前のいずれかが既にハイライト登録済み |
| このレスから読み上げ | クリック位置がレス番号要素上（§10.5 参照） |
| トグル3種 | 常時表示（✓ マークで現在状態を表示） |


### 16.4 ハイライトの適用

| ハイライト種別 | 保存先 | レス表示への反映 |
|---------------|--------|----------------|
| ID ハイライト | `id-highlights.json` | レスヘッダーの ID 部分の文字色を設定色に変更 |
| テキストハイライト | `text-highlights.json` | 本文内の該当文字列を `<span style="background:色">` で囲む |
| 名前ハイライト | `text-highlights.json` | 名前フィールドにも背景色を適用 |

- 色選択後、対応する JSON ファイルに即座に保存（IPC 経由）
- 現在表示中の全レスを即時再レンダリング
- 字幕ウィンドウにもハイライト情報を同期

### 16.5 ハイライト解除

- コンテキストメニューから「ハイライト解除」を選択
- 該当エントリを JSON から削除し、即座に再レンダリング

### 16.6 トグル項目

| 項目 | 対応設定キー | デフォルト |
|------|------------|-----------|
| オートリロード | `App.autoReloadEnabled` | true |
| オートスクロール | `App.autoScroll` | true |
| 読み上げ | `Speech.enabled` | false |

- トグル操作後、即座に設定に永続化
- ツールバーの既存 UI と状態を双方向同期

### 16.7 メニュー UI 仕様

- 絶対配置の `<div>` で実装（クリック座標に表示）
- メニュー外クリック / Escape キーで閉じる
- 画面端に近い場合は表示位置を自動調整
- サブメニュー（色パレット）は親項目のホバーで右側に展開

## 18. UI 詳細仕様（追加）

### 18.1 設定パネル

- メニューバーの「ツール」と「ヘルプ」の間に「設定」メニューを配置
- 設定パネルのレイアウト: 左にカテゴリナビ（幅110px）、右にコンテンツ（スクロール可）
- カテゴリ: 表示 / 書き込み / 読み上げ / 字幕 / プロキシ / NG / ハイライト / 情報
- NG 設定は設定パネル内の「NG」カテゴリに統合

### 18.2 スレッド一覧

- 「Since」列（スレ作成日）を「最終取得」と「勢い」の間に追加
- スレッドキー（UNIX タイムスタンプ）から `YYYY/MM/DD` 形式で表示
- 「Since」列もソート可能

### 18.3 レス表示

- 名前の右にメール列を表示: `[メールアドレス]` 形式
- sage の場合は `[sage]` をブルー系の色で表示
- 空メールは非表示

### 18.4 新着レスペイン

- スレッド一覧エリア（左パネル）の下部に配置（レスエリアからスレッドエリアへ移動）

### 18.5 アプリアイコン

- ウィンドウタイトルバーアイコン・タスクバーアイコンは `V/V.png` を使用
- バージョン情報パネルのアイコンも同様

### 18.6 バージョン情報パネル

- 「公式サイト」ボタン・リンクを削除
- 「Buy me a coffee」ボタン・リンクを削除
- 「配布先 (GitHub)」ボタンを追加（`https://github.com/kaedekiku/LiveFakeTauri2`）

### 18.7 セッション復元

- アプリ終了時（`beforeunload`）に現在表示中のスレッドの読み位置（スクロール位置）を自動保存
- 次回起動時、設定「起動時に前回のタブと板を復元」が有効な場合:
  - 前回開いていたタブを全て復元
  - アクティブタブのスレッドを最後に見ていたレス番号（または栞）までスクロール
- デフォルト: 復元有効（初期値 `true`）
